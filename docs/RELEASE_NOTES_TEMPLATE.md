# Release Notes Template

> Use this template when publishing a new version to Google Play Store.
> Play Store release notes have a **500 character limit** per language.

---

## Template

```
[Version X.Y.Z]

NEW
- [Brief description of new feature]
- [Brief description of new feature]

IMPROVED
- [What was improved and why it matters]
- [What was improved and why it matters]

FIXED
- [Bug that was fixed]
- [Bug that was fixed]

Thank you for using Dating App! Rate us if you're enjoying the experience.
```

---

## Example: Version 1.4.2 (Current)

```
What's new in 1.4.2:

NEW
- Events & meetups — discover local events to meet people in person
- Dark mode support for a comfortable experience at night

IMPROVED
- Faster profile loading and smoother swipe animations
- Better match notifications so you never miss a connection

FIXED
- Resolved occasional chat message delivery delays
- Fixed profile photo upload on some Android devices

Love the app? Leave us a review!
```

_Character count: ~390_

## Example: Bug Fix Release

```
Version X.Y.Z — Bug fixes & performance

FIXED
- Resolved crash on launch for some Android 14 devices
- Fixed notifications not appearing in the background
- Corrected distance display accuracy

IMPROVED
- Faster app startup time
- Reduced battery usage during background location updates

Thank you for your feedback — keep it coming!
```

## Example: Major Feature Release

```
Version X.Y.Z — Introducing [Feature Name]!

NEW
- [Feature Name]: [One-sentence description]
- [Secondary feature]: [One-sentence description]

IMPROVED
- Redesigned [screen/flow] for a cleaner, faster experience
- [Performance improvement]

We'd love to hear what you think. Rate us on Google Play!
```

---

## Guidelines

1. **Keep it under 500 characters** — Google Play truncates anything longer.
2. **Lead with the most exciting change** — users scan quickly.
3. **Be specific** — "Improved performance" is weak; "50% faster profile loading" is strong.
4. **Skip internal/technical jargon** — write for users, not developers.
5. **End with a call to action** — ask for a rating or feedback.
6. **Use consistent formatting** — stick to the NEW / IMPROVED / FIXED categories.
7. **Localize if needed** — Play Console supports per-language release notes.
