# Deployment Strategy & Configuration

**Last Updated:** March 4, 2026  
**Status:** Production-Ready

---

## 📋 Overview

This dating app uses a **multi-platform deployment strategy** with three separate config files:

```
Frontend (Web)    → Vercel          (vercel.json)
Backend (API)     → Render          (render.yaml)
Mobile (iOS/Andr) → EAS Build       (eas.json)
```

---

## 🚀 Deployment Platforms

### 1. **Frontend (Web) - Vercel**
**Config File:** [vercel.json](../vercel.json)  
**Platform:** Vercel  
**URL:** https://dating-app-seven-peach.vercel.app

**What it does:**
- Builds web version using: `npm run vercel-build` → outputs to `web-build/`
- Deploys React Native Web app to Vercel CDN
- Enforces security headers (CSP, HSTS, etc.)
- Rewrites all routes to `index.html` (SPA routing)

**Key Features:**
- Build: `npx expo export -p web --output-dir web-build`
- Dev: `npm start`
- Install: `npm install --legacy-peer-deps`
- Security: Strict CSP, HSTS, X-Frame-Options, XSS protection

**Deploy Trigger:** Push to main branch or manual deploy via Vercel dashboard

---

### 2. **Backend (API) - Render** 
**Config File:** [render.yaml](../render.yaml)  
**Platform:** Render  
**URL:** https://dating-app-backend-render.onrender.com

**What it does:**
- Deploys Node.js Express API server
- Runs from `backend/` directory
- Handles authentication, database queries, real-time chat
- Health check endpoint: `/health`

**Key Environment Variables:**
```
NODE_ENV=production
PORT=10000
MONGODB_URI=<set manually in Render dashboard>
FIREBASE_PROJECT_ID=my-project-de65d
FIREBASE_PRIVATE_KEY=<set manually - security sensitive>
FIREBASE_CLIENT_EMAIL=<set manually - security sensitive>
GOOGLE_CLIENT_SECRET=<set manually - security sensitive>
JWT_SECRET=<auto-generated>
```

**Deploy Trigger:** 
- Manual: Click "Deploy" in Render dashboard or use Blueprint
- Auto: Push to main branch (requires GitHub integration)

**Note:** Sensitive credentials MUST be set manually in Render dashboard, not in repo.

---

### 3. **Mobile (iOS/Android) - EAS Build**
**Config File:** [eas.json](../eas.json)  
**Platform:** EAS (Expo Application Services)  
**Platforms:** iOS App Store & Google Play Store

**Build Profiles:**
- **development** - Development client, APK/simulator
- **preview** - Internal testing, staging environment
- **production** - App Store/Play Store release

**Environments:**
```
development  → EXPO_PUBLIC_ENV=development
preview      → EXPO_PUBLIC_ENV=staging
production   → EXPO_PUBLIC_ENV=production
```

**Deploy Process:**
```bash
# Development (local testing)
eas build --platform android --profile development

# Preview (staging/internal testing)
eas build --platform all --profile preview

# Production (App Store release)
eas build --platform ios --profile production
eas build --platform android --profile production
```

## 📚 Detailed Configuration Files

### Frontend: `/vercel.json`
Deploys React Native Web app to Vercel:
- **Build:** Expo export to web-build/
- **Dev:** Expo dev server
- **Routing:** SPA routing (all paths → index.html)
- **Security:** 9 security headers configured
- **CSP:** Restricts script/style sources to self

### Backend: `/render.yaml`
Deployment blueprint for Render platform:
- **Runtime:** Node.js
- **Start:** `npm start` (runs backend/server.js)
- **Health:** `/health` endpoint monitored
- **Auto-restart:** On crash
- **Env vars:** Mix of auto-generated and manual secrets

### Mobile: `/eas.json`
Expo Application Services configuration:
- **3 profiles:** development, preview, production
- **Auto-updates:** Via Expo Updates service
- **Distribution:** App Store, Play Store, or internal
- **Resource allocation:** Varies by profile

## 🔐 Security Considerations

### Secrets Management

**DO NOT commit to repo:**
- Firebase private keys ❌
- Google Client Secrets ❌
- JWT secrets ❌
- MongoDB connection strings ❌

**Set these manually in:**
- **Render Dashboard** → Environment variables
- **Vercel Dashboard** → Environment variables  
- **EAS Build Secrets** → eas.json (credentials)

### CORS Configuration
Frontend URL in backend config:
```
CORS_ORIGIN=https://dating-app-seven-peach.vercel.app
```
Update if frontend URL changes.

## 🚨 Deployment Checklist

Before deploying to production:

- [ ] All environment variables set in platform dashboards
- [ ] Firebase project configured and accessible
- [ ] MongoDB connection tested
- [ ] API health check responding: `GET /health`
- [ ] Frontend can reach backend API
- [ ] SSL/TLS certificates valid
- [ ] Security headers present in Vercel response
- [ ] No secrets committed to git
- [ ] Tests passing: `npm test`
- [ ] Linting passing: `npm run lint`
